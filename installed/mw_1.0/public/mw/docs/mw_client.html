(static) getSubscription(name, shortDescription, description, nullopt, nullopt, nullopt) → {Object}

Create a named subscription. Clearly the creatorFunc is ignored if the subscription exists on the server already.
Parameters:
Name Type Attributes Description
name string Unique subscription name for this service.
shortDescription string a short decription. For example one or two words like: "red_truck". The server will append to this string making it more unique.
description string a longer description. May have HTML in it.
null function <optional>
 creatorFunc - creatorFunc is called if this subscription does not exist in the service yet.
null function <optional>
 readerFunc - readerFunc is if this client is subscribed and a client is writing to the subscription.
null function <optional>
 cleanupFunc - cleanupFunc is called if the subscription is destroyed on the server.

(static) getSubscriptionClass(className, shortDescription, description, nullopt, nullopt, nullopt) → {Object}

Create a unnamed subscription, as in we do not care what it is called on the server. The new subscription created by each client that calls this is just defined by the callback functions that are set.
