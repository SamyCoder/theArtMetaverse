Developer Tests

These are code development tests and are not necessarily intended to be good examples.

simple world
Loads a very simple model and that's about it. A sanity check.
move via x3d data attribute
Animates a model by adding extra javaScript code via HTML interpreting "data-" attributes as files to load. At this point it's playing with the "data-" attributes and may or may not become a Mirror Worlds common method.
example move 2
Like the above, plus loads the moving model twice. This may not be the best way to do this kind of thing, but it's a good test that show how the two animations act independently.
gnome and moving plane
Moving a different part of the model just as a test.
2 client connections in one browser client
Run another server via: 
./mw_server --http_port=8881 --https_port=8883
and then click to make 2 connections with one web client
list avatars
Lists the avatars in /mw/avatars. See the avatar X3D files in the browser debug console.
simple named subscription
The simplest subscription. You will not see anything without opening the browser console debugger. This makes subscriptions that write to console.log().
simple viewpoint avators
Test the "viewpoint avavtar" subscription with teapots.
blob avators
Test the "blob avavtars" subscription.
simple lamp
Test lamps using a subscription.
simple chat
Test simple chat using a subscription.
simple viewpoint avators, lamps, and chat All at in one page.
Test the last three subscriptions in one page.
autonomous avators, lamps, and chat All at in one page.
Test the last three subscriptions in one page with autonomous avatar.
